Administrator Guide

Configure pricing data and approve users so the quoting team always has current information.

Use this guide alongside the CSV import tools on each admin page. The help sections below are linked directly from the form labels to surface the context your team needs while editing data.

Rate tables and zones

The quote engine references multiple rate tables to calculate base charges. Keep these tables synchronized so mileage, zone, and weight break logic stays accurate:

  • Hotshot and Beyond tables: Define the mileage brackets, weight breaks, and destination zones that drive per-mile, per-pound, and fuel-percentage pricing. Align mileage breaks between the two tables so quotes switch tiers cleanly.
  • Cost and air cost zones: Map origin/destination combinations to margin buckets. Verify that the concat identifier stays unique and that each zone has an associated minimum charge for air freight.
  • ZIP zone assignments: Tie individual ZIP codes to the destination zone and note when a location requires beyond mileage. Refresh this table whenever carrier coverage shifts.

After updating any rate table, spot check a few sample quotes to confirm the correct tier triggers. Maintain a changelog so operations can trace pricing adjustments back to their effective dates.

Accessorial surcharges

Accessorial definitions add flat or percentage-based surcharges to quotes. Keep descriptions short and consistent with what appears on customer invoices. When adding a new accessorial:

  1. Choose a descriptive name that dispatch and accounting will recognize.
  2. Set the amount as either a currency value or percentage, then flag whether it should be treated as a percentage.
  3. Review the surcharge in a test quote to verify it applies once and rounds the same way invoices do.

Periodically audit the accessorial list to retire outdated fees and confirm that margin calculations still meet policy.

User approval workflow

Administrators control who can access the quote tool and which features they can use. Follow these checkpoints when approving a new account:

  • Verify the requester's company and contact details match an existing partner relationship.
  • Decide whether the user needs administrator permissions. Admins can edit rate data, approve other users, and manage accessorials.
  • Encourage strong passwords and enable multi-factor authentication when the account handles sensitive pricing data.

Update or deactivate accounts promptly when team members change roles. The audit logs on the admin dashboard highlight recent approvals and edits to help you trace changes.
